cleanup-crew

Scan AWS for unused resources, review the evidence, and clean them up safely.

scan→ review→ sweep read-only · tag-and-wait · re-verified before delete

Why

Idle EBS volumes, unassociated Elastic IPs, orphaned snapshots, and empty load balancers quietly accumulate AWS cost. cleanup-crew finds them, explains why each one was flagged, and only ever deletes something after it's been explicitly approved and a grace period has passed.

How it works

  • scan — read-only. Walks your enabled regions, flags candidates with evidence.
  • review — approve or decline each finding; approved ones get tagged cleanup:pending-deletion in AWS (reversible).
  • sweep — re-verifies each pending resource is still unused, then deletes it once its grace period has elapsed.
  • report — renders current findings to a static HTML page like this one.

v1 resource coverage

  • Unattached EBS volumes
  • Unassociated Elastic IPs
  • Orphaned EBS snapshots
  • Stopped EC2 instances (past a configurable age threshold)
  • Idle load balancers (ALB/NLB/classic ELB with no registered targets)
  • Unused security groups

Install

python3 -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
cleanup-crew scan --profile my-profile
Least-privilege IAM policy: iam-policy.json — no admin credentials required.
